# Отчёт о функциональном тестировании сайта Excursium

**Дата проведения:** 07.04.2026 – 13.04.2026  
**Тестировщик:** Антон Мороз  
**Окружение:** Windows 10, Firefox 136

---

## 1. Объект тестирования

Фронт-сайт туристической компании Excursium (`https://excursium.com/`), включая:
- лендинг,
- страницу входа/регистрации,
- страницу со списком экскурсий (фильтр),
- страницу конкретной экскурсии (онлайн-калькулятор).

---

## 2. Цель тестирования

Проверить работоспособность ключевых функций фронт-сайта, выявить дефекты и предоставить заказчику информацию о качестве продукта.

---

## 3. Что тестировалось

- Главная страница
- Регистрация и авторизация пользователей
- Фильтрация экскурсий
- Калькулятор цены на странице экскурсии
- Поиск
- Восстановление пароля

## 5. Автоматизированное тестирование

### 5.1 Инструменты

- **Selenium** + **pytest** – для автоматизации UI-тестов.
- **Page Object Model (POM)** – для структурирования кода.
- **Firefox** + **GeckoDriver** – браузер и драйвер.
- **Google Sheets** – для управления тест-кейсами.
- **Faker** – библиотека для генерации тестовых данных (email, имена и т.д.). В текущем проекте используется только потенциально, так как регистрация выполнена с реальной почтой для упрощения демонстрации. При необходимости может быть легко задействована для полной автоматизации.

### 5.2 Что покрыто автотестами

| Тест | Описание | Результат |
|------|----------|------------|
| `test_registration.py` | Регистрация нового пользователя (код подтверждения вводится вручную) | ✅ PASSED |
| `test_login.py` | Авторизация зарегистрированного пользователя | ✅ PASSED |
| `test_filter.py` | Фильтрация экскурсий по месту проведения | ✅ PASSED |
| `test_calculator.py` | Изменение цены при выборе группы «25–34» | ✅ PASSED |

### 5.3 Особенности автоматизации

- **Регистрация:** автоматически заполняется форма, отправляется запрос на получение кода. Код подтверждения вводится вручную (из email), так как сайт не принимает временные email-сервисы, а API для чтения писем отсутствует. После успешного ввода кода тест завершается.
- **Вход, фильтр, калькулятор** – полностью автоматизированы.

---

## 6. Выводы и рекомендации

- **Сайт в целом работоспособен**, основные сценарии (вход, фильтрация, калькулятор) выполняются корректно.
- **Найденные баги** (2 шт.) не являются критическими, но их рекомендуется исправить для улучшения пользовательского опыта.
- **Автотесты готовы к использованию** и могут быть интегрированы в процесс регрессионного тестирования.
- **Рекомендуется** доработать тест регистрации для полной автоматизации (например, через временные email-сервисы или API почтового ящика), однако в текущем виде он пригоден для ручной проверки с минимальным участием тестировщика.
- **Библиотека Faker** может быть задействована для генерации уникальных email, имён и других данных при необходимости.
